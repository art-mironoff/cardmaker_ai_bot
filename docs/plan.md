# План: Тестовая версия бота для генерации карточек товаров

## Контекст

Заказчик хочет Telegram-бота для генерации карточек товаров/инфографики для маркетплейсов. Перед началом полной разработки (бюджет 75к, 3 этапа) нужно сделать **минимальную тестовую версию**, чтобы:
1. Проверить, что AI-генерация даёт приемлемое качество карточек
2. Сравнить несколько AI-сервисов по качеству и цене
3. Дать заказчику потрогать результат перед оплатой

Стек: **Node.js + TypeScript + grammY** (разработчик — фронтендер).

---

## Что входит в тестовую версию

Только самый минимум для проверки гипотезы:
- Бот принимает фото + текст → отправляет в AI API → возвращает сгенерированную карточку
- Whitelist доступ (только разработчик + заказчик)
- Поддержка 2-3 AI-провайдеров для сравнения

**Что НЕ входит:** меню, навигация, оплата, баланс, мультигенерация, редактирование, справочные разделы.

## Шаги реализации

### Шаг 1: Инициализация проекта
- `npm init`, установка зависимостей: `grammy`, `typescript`, `dotenv`, `tsx`
- Настройка `tsconfig.json` (ESM, strict)
- Файл `.env` с переменными: `BOT_TOKEN`, `ALLOWED_USER_IDS`, API-ключи провайдеров
- `.gitignore` (node_modules, .env, dist)

### Шаг 2: Скелет бота
- `src/bot.ts` — инициализация grammY, подключение middleware и хендлеров
- `src/config.ts` — чтение .env, экспорт типизированного конфига
- `src/middleware/whitelist.ts` — middleware: если user ID не в списке → игнорировать
- Команда `/start` — простое приветствие

### Шаг 3: Интерфейс AI-провайдеров
- `src/providers/types.ts` — общий интерфейс:
  ```ts
  interface CardProvider {
    name: string;
    generate(imageBuffer: Buffer, prompt: string): Promise<Buffer>;
  }
  ```
- Это позволит легко переключаться между провайдерами через конфиг

### Шаг 4: Реализация провайдеров (по приоритету)

**4a. OpenAI GPT-4o** (`src/providers/openai.ts`)
- API: `POST /v1/images/generations` с input image
- GPT-4o умеет принимать изображение и генерировать новое на его основе
- Хорошо понимает русский текст в промптах
- Цена: ~$0.02-0.08 за изображение (зависит от размера)

**4b. Stable Diffusion** (`src/providers/stability.ts`)
- API: Stability AI REST API, image-to-image endpoint
- Хорош для трансформации исходного фото в стилизованную карточку
- Цена: ~$0.03 за изображение

**4c. Ideogram** (`src/providers/ideogram.ts`)
- API: Ideogram REST API, inpainting/edit endpoint
- Лучший в рендеринге текста на изображениях
- Цена: ~$0.03-0.04 за изображение

### Шаг 5: Хендлер генерации
- `src/handlers/generate.ts`:
  1. Получает сообщение с фото + caption
  2. Скачивает фото через Telegram API (`ctx.getFile()`)
  3. Формирует промпт для AI (системный промпт + пользовательский текст)
  4. Вызывает текущий провайдер
  5. Отправляет результат пользователю как фото
  6. При ошибке — понятное сообщение
- Системный промпт (базовый): "Создай карточку товара для маркетплейса на основе этого изображения. Используй следующее описание: {user_text}. Карточка должна быть профессиональной, с красивым оформлением."

### Шаг 6: Команда переключения провайдера (для тестирования)
- Команда `/provider <name>` — переключает AI-провайдер на лету
- Только для whitelist-пользователей, чтобы быстро сравнивать результаты

## Промпт-инжиниринг

Ключевая часть — системный промпт для AI. Базовая стратегия:
- Описать роль: "Ты — дизайнер карточек для маркетплейсов"
- Указать что на входе: фото товара + описание
- Указать что на выходе: стилизованная карточка с текстом
- Текст в кавычках из пользовательского сообщения — использовать как есть
- По умолчанию формат 3:4 (самый популярный для маркетплейсов)

Промпт будет уточняться итеративно по результатам тестов.

## Проверка работоспособности

1. Запустить бота: `npx tsx src/bot.ts`
2. Написать боту в Telegram `/start` — должен ответить приветствием
3. Отправить фото товара с описанием — должен вернуть сгенерированную карточку
4. Переключить провайдер: `/provider stability` — повторить генерацию
5. Сравнить результаты разных провайдеров по качеству и скорости
6. Проверить whitelist: написать с другого аккаунта — бот не должен отвечать
